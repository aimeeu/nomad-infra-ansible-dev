# Consul Single Machine

A single-node Consul server deployment using Multipass for local development and testing.

## Overview

This configuration deploys a standalone Consul server in a Multipass VM. It's ideal for:
- Learning Consul basics
- Testing Consul configurations
- Development environments requiring service discovery
- Integration testing with Nomad or other HashiCorp tools

## Architecture

- **1 Consul Server**: Single-node Consul cluster with UI enabled
- **VM Resources**: 2 CPUs, 2GiB RAM
- **Datacenter**: lcy1 (London)
- **Features**: 
  - Consul UI enabled
  - Consul Connect (service mesh) enabled
  - Bootstrap mode (single server)

## What the Terraform Does

[`main.tf`](main.tf:1) provisions the infrastructure:

1. **Creates Multipass VM** (via `multipass-compute` module):
   - Instance name: `consul-server-<random-id>`
   - Specs: 2 CPUs, 2GiB RAM
   - SSH key: Uses your `~/.ssh/id_rsa.pub`
   - Ansible group: `consul_server`

2. **Runs Ansible Provisioning** (via `ansible-provision` module):
   - Executes [`playbook_all.yaml`](playbook_all.yaml:1)
   - Configures Consul server
   - Outputs SSH command and Consul UI URL

3. **Outputs**:
   - SSH connection command
   - Consul UI URL (http://IP:8500)
   - Consul server IP address

## What the Ansible Playbooks Do

### [`playbook_all.yaml`](playbook_all.yaml:1)
Master playbook that orchestrates the deployment:
- Imports [`playbook_consul_server.yaml`](playbook_consul_server.yaml:1)
- Imports `playbook_localhost.yaml` (local configuration)

### [`playbook_consul_server.yaml`](playbook_consul_server.yaml:1)
Configures the Consul server VM:

1. **Common Role**: Sets up base system
   - Sets hostname
   - Installs packages: build-essential, git, jq, make, net-tools, unzip

2. **Consul Role**: Installs and configures Consul
   - Version: 1.22.6
   - Binary checksum verification
   - Config directory: `/etc/consul.d/`
   - Data directory: `/opt/consul/data/`
   - Uses template: [`consul_server.hcl.j2`](templates/consul_server.hcl.j2:1)

### Consul Configuration ([`consul_server.hcl.j2`](templates/consul_server.hcl.j2:1))

Key configuration settings:
- **Log Level**: DEBUG (for development)
- **Datacenter**: lcy1
- **Server Mode**: Enabled with bootstrap_expect=1
- **UI**: Enabled and accessible
- **Addresses**: Binds to VM's default IPv4 address
- **Consul Connect**: Enabled for service mesh
- **Retry Join**: Configured for cluster formation (supports multi-node expansion)

## Prerequisites

- **Multipass**: Install from [multipass.run](https://multipass.run)
- **Terraform**: v1.0 or later
- **Ansible**: v2.9 or later
- **SSH Key**: `~/.ssh/id_rsa.pub` must exist

## Deployment Instructions

### 1. Initialize Terraform

```bash
cd lab/multipass/consul-single-machine
terraform init
```

### 2. Review the Plan

```bash
terraform plan
```

### 3. Deploy

```bash
terraform apply
```

Type `yes` when prompted. Deployment takes 2-5 minutes.

### 4. Access Consul

After deployment completes, Terraform outputs:
```
SSH:        ssh 192.168.64.X
Consul UI:  http://192.168.64.X:8500
```

**Access the Consul UI**: Open the URL in your browser

**SSH into the VM**:
```bash
ssh 192.168.64.X
```

### 5. Verify Consul

Check Consul status:
```bash
ssh 192.168.64.X "consul members"
```

Expected output:
```
Node                    Address          Status  Type    Build   Protocol  DC    Partition  Segment
consul-server-abc123    192.168.64.X:8301  alive   server  1.22.6  2         lcy1  default    <all>
```

## Usage Examples

### Register a Service

SSH into the VM and create a service definition:
```bash
ssh 192.168.64.X

cat > /etc/consul.d/web.json <<EOF
{
  "service": {
    "name": "web",
    "port": 8080,
    "check": {
      "http": "http://localhost:8080/health",
      "interval": "10s"
    }
  }
}
EOF

consul reload
```

### Query Services via DNS

```bash
dig @192.168.64.X -p 8600 web.service.consul
```

### Query Services via HTTP API

```bash
curl http://192.168.64.X:8500/v1/catalog/services
```

## Troubleshooting

### VM Not Starting
```bash
multipass list
multipass info consul-server-*
```

### Consul Not Running
```bash
ssh 192.168.64.X "sudo systemctl status consul"
ssh 192.168.64.X "sudo journalctl -u consul -f"
```

### Ansible Fails
Check Ansible logs in Terraform output. Common issues:
- SSH key not found: Ensure `~/.ssh/id_rsa.pub` exists
- Network connectivity: Verify Multipass networking

### Cannot Access UI
- Verify Consul is running: `systemctl status consul`
- Check firewall rules
- Ensure you're using the correct IP from Terraform output
